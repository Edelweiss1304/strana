Этот документ создан для того, чтобы любой пользователь (который что-то в этом понимает) смог разобраться в коде и тестах. Здесь будут объяснены назначения каждой папки и файла, для легкого погружения (на всякий случай)

Документ будет постоянно обновляться и дополняться.

1. Папка "base".

Здесь расположен файл base_class, который содержит в себе классы, применяющиеся очень широко. Например, функция для создания скриншота или проверки слова на странице. Все функции я буду подписывать комментариями.
Главное понять, какие именно функции содержаться в этом файле (еще там лежат URL главных страниц).

2. Папка "pages"

В папке "pages", как уже понятно из названия, хранятся файлы с названиями страниц сайта. Внутри этих файлов можно найти локаторы(#Locators), Геттеры(#Getters), Действия(#Actions),Методы(#Methods).
Они разделены между собой комментариями, где какая категория начинается довольно понятно.

#Locators - Нужны для хранения локатора элемента на веб-странице. Если вдруг локатор меняется, то его нужно будет поменять только в этом месте.
#Getters - Туда подставляется локатор, чтобы с помощью селениума можно было получить веб-элемент.
#Actions - Различные одинарные действия на странице, с использованием вышеупомянутых геттеров. Например, на кнопку можно и нажать и спарсить с нее текст. Эти два действия записываются отдельно.
#Methods - Набор из нескольких действий. Потом из этих методов составляются сами тесты.

3. Папка "Screen"

Сюда попадают скриншоты после тестов(если это необходимо)

4. Папка "tests"

Здесь хранятся сами тесты. По названию теста в принципе понятно, что он делает.

5. Папка "utilities"

Здесь содержится файл Conftest, который хранит в себе фикстуры, дабы не писать их в каждом тестовом файле.
